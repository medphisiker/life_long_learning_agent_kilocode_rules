# Версионирование и Окружение (Versions & Environment)

Для обеспечения стабильности и воспроизводимости среды разработки установлены следующие стандарты версий.

## Глобальные стандарты

Эти требования применяются ко всем сервисам, **КРОМЕ** `rag` и `test_generator` (они управляются отдельными командами).

| Инструмент | Версия | Примечание |
| :--- | :--- | :--- |
| **Python** | `3.13` | Базовая версия для всех новых сервисов и скриптов. |
| **uv** | `0.9.26` | Пакетный менеджер. Использовать `pip install uv==0.9.26` в Dockerfile. |

## Исключения

Сервисы `rag` и `test_generator` могут использовать другие версии Python и пакетных менеджеров, зафиксированные в их `Dockerfile` и `pyproject.toml`. Изменять их конфигурацию без явного запроса **ЗАПРЕЩЕНО**.

## Dockerfile Best Practices

Для соблюдения стандартов используйте следующий шаблон установки `uv`:

```dockerfile
FROM python:3.13-slim-bookworm

# Фиксация версии uv
RUN pip install uv==0.9.26

WORKDIR /app
# ...
```

## Проверка версий

При начале работы над сервисом (кроме исключений) убедитесь, что в `Dockerfile` и `pyproject.toml` (requires-python) указаны корректные версии.
