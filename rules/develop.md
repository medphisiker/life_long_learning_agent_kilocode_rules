# Правила разработки (Development Rules)

Данный документ определяет стандарты и подходы к разработке, отладке и тестированию в проекте.

Отвечай на русском языке.

## 1. Конфигурация и Docker
- **Версионность образов**: Всегда указывай точные версии (tags) для готовых Docker-образов. Использование тега `latest` запрещено.
- **Стандарты версий**: См. [`tech_stack.md`](tech_stack.md).
- **Сетевое взаимодействие**: Все сервисы должны использовать именованные внешние сети.

## 2. Принципы работы с кодом
- **Объективность**: Не предполагай поведение системы, исследуй код с помощью `codebase_search`.
- **Верификация**: Проверяй реализацию через `codebase_search` перед изменениями.
- **Актуальность**: Сверяйся с документацией через `context7`.
- **Исследование**: Используй `tavily` для внешних решений.

## 3. Тестирование
- **Единый центр**: Все команды запуска тестов собраны в [`testing.md`](testing.md).
- **WebSocket**: Проверяй последовательность событий (steps/tools) через WebSocket.

## 4. NetRunner Protocol
- Спецификация взаимодействия вынесена в [`netrunner_protocol.md`](netrunner_protocol.md).

## 5. Документирование
- Изменения в логике графа -> `agent_service/docs/agent_documentation.md`.
- Новые идеи по UI/UX -> `docs/ideas/`.
