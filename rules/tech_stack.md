# Технологический Стек и Стандарты (Tech Stack)

> **CORE**: Этот файл определяет базовые технологии и версии для всех сервисов проекта.

## 1. Языки и Пакетные менеджеры

| Инструмент | Версия (Стандарт) | Примечание |
| :--- | :--- | :--- |
| **Python** | `3.13` | Базовая версия для всех новых сервисов (`agent_service`, `web_ui_service`, `user_service`). |
| **uv** | `0.9.26` | Пакетный менеджер. Использовать `pip install uv==0.9.26` в Dockerfile. |
| **Node.js** | `LTS` | Для фронтенда (React). |
| **LLM (Agent)** | `Gemini 3 Flash` | Основная модель через `ai-mediator.ru`. |

### Исключения
Сервисы `rag` и `test_generator` могут использовать другие версии (см. их `Dockerfile`), так как управляются отдельными командами.

## 2. Docker и Инфраструктура

*   **Базовые образы**: `python:3.13-slim-bookworm` (где возможно).
*   **Теги**: Всегда указывать точные версии (например, `postgres:15.10-alpine`). `latest` запрещен.
*   **Сети**: Использовать именованные внешние сети (`external: true`) для связи между контейнерами.

## 3. Принципы разработки

*   **Codebase First**: Не предполагай — проверяй код через `codebase_search`.
*   **Verification**: Любое изменение должно быть проверено (тесты или запуск).
*   **Documentation**: Обновляй Memory Bank (`.kilocode/rules/`) после завершения задачи.

*   **Стиль**: "Нетраннерская Дека" (Cyberpunk HUD).
